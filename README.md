# 零售便利店平台笔试 Demo

本项目基于 `React + TypeScript + Ant Design 5.x` 实现，聚焦同一套 `零售便利店平台` 下的四个核心迭代需求：

- `点位申请支持“新店转老店升级”`
- `员工维护中保护法人账号，避免误删/误停用`
- `校园场景下订单统计受寒暑假影响波动极大，导致数据失真`
- `运营人员在查看点位申请记录时，无法快速跳转到对应的审批详情`
当前界面执行口径为：`严格按 Ant Design 5.x 官方默认组件风格落地`，不使用业务系统二次主题化皮肤。

## Demo 覆盖范围

- 左侧统一平台导航，纳入 `申请记录`、`员工维护`、`校园经营分析`、`审批管理` 四个模块
- 申请记录列表页条件展示 `转老店升级`
- 从状态入口点击 `点位信息待完善 -> 弹窗 -> 新店申请详情`
- 转换确认弹窗
- 地址一致 / 不一致校验
- 关联门店地址为空的继续流转
- 无可选门店空状态
- 门店资料缺失后的表单补录
- 老店升级表单自动带出字段
- 提交成功后生成新记录、关闭原记录、释放点位的结果反馈
- 员工维护页中对 `法人账号` 的显式标记
- 法人账号默认禁用 `停用 / 删除`
- `保护规则 -> 发起移交 -> 新法人账号接替` 的演示闭环
- 校园经营分析页提供 `按日历月看` 与 `剔除假期影响后看` 两种视角
- 支持 `标准校正场景`、`校历缺失场景` 与 `官网导入后生效`
- 主页面通过 `校历管理` 统一查看校历来源、连续假期区间、官网导入结果与手动调整入口
- 在校历缺失场景中可通过 `导入学校官网校历` 触发导入并立即重算统计
- 审批管理页支持从点位申请记录直接打开审批详情抽屉
- 对 `审批详情待同步` 的记录给出明确禁用态和状态提示

## 演示建议

进入页面后，建议按以下顺序演示：

1. `申请记录`
2. `员工维护`
3. `校园经营分析`
4. `审批管理`

其中 `申请记录` 模块可继续在右侧 `Demo 场景切换` 中演示 `标准成功流 / 门店地址为空 / 无可选门店 / 门店资料缺失`。这样可以在一套 Demo 中完整展示正常流、异常流和边界状态。
